**未登录高画质、只读评论与播放器增强，一份脚本集中解决。**

|---|---|---|
| **画质解锁** | 1080P / 720P / 480P / 360P | SPA 切视频、分 P 和画质掉档时自动恢复首选画质 |
| **播放器增强** | 预设倍速、自定义倍速、前进后退、临时加速 | 登录与未登录均可使用，设置自动保存 |

| **观看防打扰** | 登录遮罩处理、自动暂停防护、静音意图保护 | 保留用户主动暂停和站内初始播放状态 |
| **直播分区兜底** | 原接口检测、备用接口转换、保守翻页 | 分区列表异常或疑似裁剪时继续加载 |

## 快速使用

### 调整画质与功能开关

打开方式：

1. 在 Tampermonkey/Violentmonkey 菜单中点击 **画质设置**。
2. 视频播放器设置菜单中出现 **脚本设置 >** 时，也可从这里打开。

画质、协议模式、评论和直播开关在修改时立即保存。切换协议模式、评论模式或直播兜底后刷新页面，可完整重建对应模块。

### 调整倍速

播放器右侧控制栏会出现 **▶ 1x ▾** 按钮：

- 点击倍速值，选择 `0.5x` 至 `3x` 的预设速度。
- 在自定义输入框设置 `0.07x-16x`，点击 **应用**。
- 关闭 **启用倍速强制** 会将当前媒体恢复到 `1x`；已保存的倍速仍会保留，重新开启后继续使用。
- 标准浏览器全屏时倍速按钮自动隐藏，退出全屏后恢复。

### 前进、后退与临时加速

1. 点击播放器画面，显示左右圆形按钮，按钮会在 2 秒后自动隐藏。
2. 点击左侧按钮后退，默认 `10` 秒。
3. 点击右侧按钮前进，默认 `15` 秒。
4. 按住右侧前进按钮 2 秒进入临时倍速，松开恢复长按前的实际速度。

前进和后退可分别设置为 `1-300` 秒，临时倍速可设置为 `1x-16x`。移动超过 12px、视频暂停、窗口失焦或页面隐藏时，临时倍速会自动取消并恢复。

### 浏览评论

- 评论功能默认开启，进入支持的详情页后自动替换为脚本的只读评论列表。
- 顶部可切换 **最热** 和 **最新** 排序。
- 默认使用无限滚动；开启分页模式并刷新后，可使用上一页、下一页、页码输入和 Enter 跳转。
- 主评论每批加载 20 条，子回复每批加载 10 条；点击回复数量可展开全部子回复。
- 直播间和用户空间中的动态评论入口会跳转到动态或 Opus 详情页，再查看完整评论。

评论模块只提供浏览、排序、翻页和展开回复，不包含发布评论、回复、点赞、举报或删除功能。跳转到较大的未访问页码时，需要逐页取得 API cursor，等待时间会更长。

> [!NOTE]
> 评论功能定位为未登录浏览工具，所有交互均为只读操作。

### 使用直播分区兜底

保持 **直播分区连续加载** 开启，正常滚动直播分区列表即可。脚本优先使用 B 站原接口，只在请求失败、分页信息矛盾或结果疑似被未登录状态裁剪时调用备用接口。

## 设置说明

### 画质设置面板

| 设置 | 默认值 | 说明 |
|---|---:|---|
| 首选画质 | 1080P | 可选 1080P、720P、480P、360P |
| 协议级解锁 | 开启 | 关闭并刷新后切换到客户端兼容模式 |
| 切换时暂停 | 关闭 | 仅客户端兼容模式切换试用画质时使用 |
| 解锁全部评论 | 开启 | 控制支持页面的只读评论重绘 |
| 分页加载评论 | 关闭 | 关闭时使用无限滚动，切换后刷新生效 |
| 直播分区连续加载 | 开启 | 控制未登录直播分区列表接口兜底 |

### 倍速弹层

| 设置 | 默认值 | 范围 |
|---|---:|---:|
| 播放倍速 | 1x | 0.07x-16x |
| 自定义倍速 | 1.5x | 0.07x-16x |
| 后退秒数 | 10 秒 | 1-300 秒 |
| 前进秒数 | 15 秒 | 1-300 秒 |
| 长按临时倍速 | 2x | 1x-16x |
| 启用倍速强制 | 开启 | 开 / 关 |

所有设置都会持久保存，并在 SPA 切视频或播放器重建后继续使用。当前版本未提供一键恢复默认设置。

## 页面与登录状态

| 页面 | 未登录功能 |
|---|---|
| `www.bilibili.com/video/*` | 画质解锁、评论、播放控制、防登录提示、防自动暂停 |
| `www.bilibili.com/list/*` | 画质解锁、评论、播放控制、防登录提示、防自动暂停 |
| `www.bilibili.com/festival/*` | 画质解锁、评论、播放控制、防登录提示、防自动暂停 |
| `www.bilibili.com/opus/*` | 只读评论、防登录提示 |
| `t.bilibili.com/<数字ID>` | 只读评论、防登录提示 |
| `www.bilibili.com/read/cv*` | 只读评论、防登录提示 |
| `space.bilibili.com/*` | 动态评论入口跳转、防登录提示 |
| `live.bilibili.com/*` | 直播分区加载兜底、动态评论入口跳转、防登录提示 |

登录后会停用画质解锁、评论重绘、直播分区兜底、防登录提示和防自动暂停；播放器倍速、前进后退、长按临时倍速继续可用。直播页和空间页的动态评论入口跳转逻辑也会保留。

## 工作原理

<details>
<summary><strong>展开查看技术实现</strong></summary>

- **协议级解锁**：默认注入会话级 `DedeUserID`，清理低画质 SSR 数据，对 `playurl` 请求按首选画质进行 WBI 重签，并从可播放响应中选择完整媒体流。
- **客户端兼容**：关闭协议级解锁并刷新后，使用自动点击高清试用、延长试看计时和画质掉档恢复作为备用路径。
- **评论读取**：直接调用 B 站评论 API 并完成 WBI 签名，在安全的评论容器内渲染只读列表。
- **播放器控制**：通过媒体倍速 setter、`ratechange`、播放器重挂载和定时校验保持用户设置。
- **直播兜底**：检测直播分区原接口结果，异常时转换备用接口数据为播放器所需结构。

真实登录状态通过带签名的 `DedeUserID__ckMd5` 识别。协议级解锁写入的会话级 `DedeUserID` 只用于未登录画质请求，不会让脚本进入真实登录分支。

</details>

## 使用说明与限制

<details>
<summary><strong>展开查看兼容性与边界</strong></summary>

- B 站播放器 DOM 或 API 调整后，控制按钮、评论容器或接口拦截可能需要同步适配。
- 协议请求未获得完整媒体流时会回退页面原请求，实际可用画质仍受视频源、地区和 B 站接口响应影响。
- 防自动暂停会识别点击、触摸、Space 和 K 键等常见用户操作，并只在视频已经播放后处理意外暂停。
- 评论接口、WBI key 或安全容器获取失败时，脚本会跳过评论重绘并在控制台记录原因。
- 直播兜底专门处理直播分区列表接口，范围聚焦分区列表连续加载。
- 倍速与跳转控件依赖 B 站当前播放器结构；直播流的跳转效果取决于播放器是否提供 DVR 时间轴。

</details>
